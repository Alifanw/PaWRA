## ‚ùì Frequently Asked Questions

### Q: Will this break my existing system?
**A**: No! 100% backward compatible. Zero breaking changes. Same database, same API, same backend.

### Q: Can I switch back to the old UI?
**A**: Yes! Just run `doorlock_ui_modern.py` instead. Both files can coexist.

### Q: Does it work on Raspberry Pi?
**A**: Yes! Tested and optimized for Raspberry Pi 3/4+. Responsive design works great on small screens.

### Q: Can I customize the colors?
**A**: Yes! Edit ThemeManager colors in the source code. Very easy to change.

### Q: Does dark mode drain battery?
**A**: No, OLED panels aren't common in Raspberry Pi touchscreens. Dark mode is for eye comfort.

### Q: How do I implement the reports?
**A**: The stub functions are ready: `_report_daily()`, `_report_monthly()`, etc. Fill them with your logic.

### Q: Can I run both UIs at the same time?
**A**: Yes, on different ports or terminals. Same backend supports both.

### Q: Is the database modified?
**A**: No! Database schema is completely unchanged. No migrations needed.

### Q: Do I need to install new packages?
**A**: No! Uses only: Tkinter (built-in), Flask, MySQL connector (already installed).

### Q: How do I revert?
**A**: Delete the new file or comment it out. Use the old `doorlock_ui_modern.py`. One-line change.
